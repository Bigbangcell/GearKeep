# 需求文档 (PRD)：个人摄影器材与电子产品管理 App

## 1. 项目概述

- **项目名称**：GearKeep (暂定)
- **目标平台**：Android (Target SDK 34+)
- **核心目标**：为摄影师和数码爱好者提供纯本地、隐私安全的器材归档工具。支持器材配套逻辑、保修追踪及带附件的本地数据迁移。
- **核心原则**：纯本地运行、无广告、数据完全由用户控制。

## 2. 技术栈约束 (Tech Stack)

- **语言**：Kotlin
- **UI 框架**：Jetpack Compose (Material 3 规范)
- **数据库**：Room Persistence Library (SQLite)
- **图片处理**：Coil (图片加载), Scoped Storage (分区存储管理)
- **OCR 库**：Google ML Kit (用于序列号识别)
- **后台任务**：WorkManager (用于保修过期检查通知)
- **数据交换**：JSON + Zip (用于本地导出导入)

## 3. 数据模型 (Data Model)

### 3.1 核心实体：Item (物品)

- `id`: UUID (主键，确保多端合并时的唯一性)
- `name`: String (物品名称)
- `brand`: String / `model`: String (品牌与型号)
- `sn`: String (序列号)
- `purchasePrice`: Double (购入价格)
- `purchaseDate`: Long (购入日期时间戳)
- `warrantyMonths`: Int (保修月数)
- `warrantyDeadline`: Long (保修截止日，自动计算)
- `location`: String (存放位置，如：1号防潮箱)
- `notes`: String (备注)
- `isMaster`: Boolean (是否为主设备，如机身)

### 3.2 关系模型：Relationship

- **方案 B (灵活关联)**：采用自关联表，存储 `parentId` 和 `childId`。
- **逻辑**：允许一个物品关联多个“配件”（如：R5机身关联 3 块电池、2 个镜头）。

## 4. 功能需求 (Functional Requirements)

### 4.1 智能录入模块

- **手动录入**：标准表单。
- **文本粘贴识别**：提供一个 Text Area。Agent 需实现一个正则表达式解析器，识别包含 `品牌`、`型号`、`价格` 的文本并自动填充表单。
- **S/N OCR 识别**：调用摄像头，利用 ML Kit 识别器材上的字符并填入序列号字段。
- **注意**：**禁用** EXIF 自动识别功能。

### 4.2 资产与保修管理

- **保修看板**：主页显示即将过保（30天内）和已过保的器材列表。
- **价值统计**：自动汇总当前所有物品的总价值。
- **本地通知**：在保修到期前 7 天发送系统通知。

### 4.3 器材配套视图

- 在物品详情页，直观展示“该设备的配套附件”列表。
- 支持“添加现有物品为附件”和“快捷解绑”。

### 4.4 增强型本地同步 (核心开发难点)

- **导出逻辑**：将 Room 数据库中的结构化数据转化为 `data.json`，并与 `images` 文件夹一同打包为 `.gearbak` (Zip格式)。
- **导入逻辑**：
  1. 解压并读取 `data.json`。
  2. **冲突处理 (Conflict Resolution)**：当检测到导入的 `id` 在本地已存在时，弹出 UI 对话框提供三个选项：
  - **覆盖替换**：使用导入的数据更新本地记录。
  - **保留本地**：忽略当前冲突项。
  - **设为新增**：生成新 UUID，作为新记录插入（允许共存）。

## 5. UI/UX 页面规划

1. **首页 (Dashboard)**：总资产统计、保修临期卡片、最新添加列表。
2. **器材库 (Inventory)**：支持分类浏览（相机、镜头、配件）和关键字搜索。
3. **添加/编辑页**：表单设计，包含 OCR 按钮和粘贴识别入口。
4. **详情页**：展示参数、保修倒计时条、关联配套物品列表。
5. **设置页**：导入/导出功能入口，清除数据选项。

## 6. Agent 引导说明 (Prompt Tips)

- *“请使用 Repository 模式处理数据层，并确保所有的 UI 逻辑都在 ViewModel 中实现。”*
- *“在处理导出导入功能时，请确保文件流关闭正确，并使用 Android 13 的媒体权限规范。”*
- *“导入冲突 UI 必须清晰显示两个版本的‘最后修改时间’供用户对比。”*
- 其他未尽事宜与用户沟通
